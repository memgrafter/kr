# Update Registry struct with kr_folder, refactor load_registry

## Notes

**2026-06-24T17:48:06Z**

Registry struct adds #[serde(skip)] kr_folder: PathBuf. load_registry returns (Registry, PathBuf) or we set kr_folder after deserialization from the file path.

Key change: registry_path() and load_registry() must return/expose the kr_folder so callers know which .kr folder the registry came from. This is critical for multi-folder mode where the same registry name could exist in different .kr folders.
